# Visión General del Proyecto

Resumen rápido de la plataforma de gestión de flotas antes de recorrer los capítulos completos.

## Propósito

Entregar un stack integral (backend, frontend, infraestructura y documentación) que demuestre cumplimiento total del desafío de Aivacol.

## Pilares

- **Arquitectura** — revisa [Arquitectura Backend](./03-arquitectura-backend.md) y [Modelado de Datos y Dominio](./04-modelado-dominio.md) para conocer módulos, capas y entidades.
- **Seguridad y observabilidad** — [Seguridad, Auditoría y Mensajería](./05-seguridad-auditoria-mensajeria.md) detalla RBAC, auditoría y eventos RabbitMQ.
- **Experiencia digital** — [Frontend y Experiencia](./06-frontend.md) cubre el patrón FSD, los temas naranjas y el flujo de datos.
- **Calidad y operaciones** — [Calidad y Pruebas](./08-calidad-pruebas.md) y [Runbook y Resolución de Problemas](./09-runbook-resolucion.md) muestran las rutinas de QA y soporte.

## Resultados

- APIs NestJS con DDD, caché Redis y eventos RabbitMQ.
- Frontend Next.js responsivo que comparte esquemas Zod con el backend.
- Stack Docker Compose con SQL Server, Redis, RabbitMQ y MongoDB.
- Sitio GitPageDocs trilingüe alineado con el desafío.
