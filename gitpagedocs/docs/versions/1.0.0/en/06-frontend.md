# Frontend and Experience

The web client uses **Next.js 14 (App Router)** and the **Feature-Sliced Design (FSD)** pattern to keep concerns modular and reusable.

## FSD structure

- `app/` — routing, global layout and styles.
- `entities/` — domain contracts (Vehicle, Brand, Model), shared UI (e.g. `vehicle-table`) and API hooks.
- `features/` — isolated features such as vehicle forms, management flows and filter bars.
- `processes/` — cross-cutting logic (authentication flows, Redux store, theme provider).
- `shared/` — UI kit, utilities, Axios client, token storage and Zod schemas exported from the backend.
- `widgets/` — composite UI blocks (AppShell, dashboard panels, reference boards).

## State & data handling

- **Redux Toolkit** stores authentication state (`auth` slice) and exposes typed hooks.
- **React Query** powers data fetching/mutations (`useVehiclesQuery`, `useCreateVehicleMutation`, etc.) via Axios.
- **HTTP client** automatically attaches the JWT and handles 401 resets.

## Shared validation

- Zod schemas are generated by the backend and imported into the frontend (`frontend/src/shared/schemas`).
- Helpers in `shared/lib/zod-helpers` ensure consistent error messages.

## Orange theming & responsiveness

- Dual light/dark orange palettes defined in `globals.css`, applied by `ThemeProvider`.
- `NEXT_PUBLIC_START_THEME` lets deployments choose the default theme (light or dark) without recompiling.
- Components were refactored for mobile-first layouts: content is centered on small screens and aligned side-by-side on larger viewports.

## UX & accessibility

- Buttons, tables and forms use semantic markup and provide feedback (`InlineMessage`, disabled states).
- Mobile navigation mirrors desktop actions, preserving the same routes and options.
- Input components enforce uppercase plates, masked hints and descriptive error text.

## Testing & linting

- Unit tests with React Testing Library (`frontend/tests/unit`).
- Playwright e2e scenarios under `frontend/tests/e2e` validate the login page and shell rendering.
- ESLint (Next.js preset + custom rules) runs via `npm run lint`.

The result is a responsive, theme-aware interface that consumes the same validation rules as the backend, providing a polished operator experience.
